# Prompt Library C App MVP

Prompt Library is a pure C command-line application for saving, organizing,
retrieving, and improving prompt templates in a local file-backed library.

## Goals

- Store prompts in a local folder users control.
- Organize prompts by folder, category, and tags.
- Retrieve prompts quickly by id, title, folder, category, tag, or text search.
- Preserve prompt history when creating optimized versions.
- Keep prompt files human-readable and portable.
- Make the CLI useful in scripts by supporting raw prompt output.

## Non-Goals For The MVP

- No network sync.
- No hosted service or database server.
- No graphical interface.
- No model-provider integration for automatic optimization.
- No multi-user locking beyond safe local file writes.

## Command Surface

### `pp init`

Initializes a prompt library root.

Options:

- `--root <path>`: create or validate a specific library root.

If `--root` is omitted, `PROMPTLIB_ROOT` is used. If that is unset, the command
uses auto-discovery (walk-up from cwd), falling back to the user's home directory.

**Status:** Implemented | **SRS:** REQ-F-009–011

---

### `pp add`

Saves a new prompt.

Options:

- `--title <text>`: required prompt title.
- `--body <text>`: prompt body inline.
- `--editor`: open `$EDITOR` to enter prompt body.
- `--folder <path>`: logical folder inside the library (default: `inbox`).
- `--category <name>`: category name (default: `general`).
- `--description <text>`: optional summary.
- `--root <path>`: library root.
- `--tag <name>`: repeatable tag.

Prompt IDs are generated by slugifying the title and appending a 4-hex-char
hash suffix (djb2).

**Status:** Implemented | **SRS:** REQ-F-012–021

---

### `pp list`

Lists prompts as a colored table.

Options:

- `--root <path>`: library root.
- `--folder <path>`: filter by folder.
- `--category <name>`: filter by category.
- `--tag <name>`: filter by tag.
- `--json`: output in JSON format.
- `--no-pager`: disable automatic paging.

**Status:** Implemented | **SRS:** REQ-F-022–029

---

### `pp show <id-or-title>`

Displays a prompt's metadata and body.

Options:

- `--root <path>`: library root.
- `--raw`: print only the prompt body.
- `--json`: output in JSON format.

**Status:** Implemented | **SRS:** REQ-F-030–033

---

### `pp edit <id-or-title>`

Updates prompt metadata or content. Without flags, opens `$EDITOR` with the
current body.

Options:

- `--body <text>`: replace body inline.
- `--category <name>`: replace category.
- `--description <text>`: replace description.
- `--root <path>`: library root.
- `--tag <name>`: replace all tags; repeatable.
- `--title <text>`: replace title.
- `--folder <path>`: move to different folder.

**Status:** Implemented | **SRS:** REQ-F-034–039

---

### `pp delete <id-or-title>`

Archives a prompt (moves to `archive/` directory).

Options:

- `--yes`: confirm the operation.
- `--permanent`: (planned) permanently remove the prompt file.

**Status:** Implemented | **SRS:** REQ-F-040–043

---

### `pp archive`

Manages archived prompts.

Actions:

| Action | Description |
|--------|-------------|
| `list` | List all archived prompts with ID, title, and archived date |
| `restore <id>` | Restore archived prompt to original folder (or `inbox`) |
| `purge <id> --yes` | Permanently delete archived prompt from filesystem |

**Status:** Planned | **SRS:** REQ-F-125–127

---

### `pp search <query>`

Searches title, body, description, category, folder, and tags (case-insensitive).

Options:

- `--folder <path>`: pre-filter by folder.
- `--category <name>`: pre-filter by category.
- `--tag <name>`: pre-filter by tag.
- `--raw`: print raw bodies for matching prompts.
- `--json`: output in JSON format.
- `--root <path>`: library root.

**Status:** Implemented | **SRS:** REQ-F-044–047

---

### `pp browse`

Interactive prompt browser. Uses `fzf` for fuzzy-search with preview when
available, falls back to a numbered menu with view/edit/quit shortcuts.

Options:

- `--folder <path>`: pre-filter.
- `--category <name>`: pre-filter.
- `--tag <name>`: pre-filter.

**Status:** Implemented | **SRS:** REQ-F-048–051

---

### `pp optimize <id-or-title>`

Creates versioned improvements without overwriting the original.

Options:

- `--body <text>`: optimized prompt body.
- `--note <text>`: rationale or change note.
- `--promote`: make the optimized version current.
- `--history`: list all versions.
- `--compare <version>`: compare current body with a version snapshot.
- `--root <path>`: library root.

Version numbers auto-increment (`0002`, `0003`, …); implicit v1.

**Status:** Implemented | **SRS:** REQ-F-052–058

---

### `pp folder`

Manages folder names in the library.

Actions:

| Action | Description |
|--------|-------------|
| `list` | Print registered folder names |
| `create <name>` | Add a new folder (idempotent) |
| `remove <name> --yes` | Remove folder (blocked if prompts reference it) |
| `rename <name> --to <new>` | Rename folder, update all referencing prompts and directories |

**Status:** Implemented | **SRS:** REQ-F-059–065

---

### `pp category`

Manages category names in the library.

Actions:

| Action | Description |
|--------|-------------|
| `list` | Print registered category names |
| `create <name>` | Add a new category (idempotent) |
| `remove <name> --yes` | Remove category (blocked if prompts reference it) |
| `rename <name> --to <new>` | Rename category, update all referencing prompts |

**Status:** Implemented | **SRS:** REQ-F-066–070

---

### `pp tag`

Incremental tag management. Unlike `pp edit --tag` (which replaces all tags),
these commands operate on individual tag values.

Actions:

| Action | Description |
|--------|-------------|
| `add <id> <tag>` | Append a single tag (no-op if present) |
| `remove <id> <tag>` | Remove a single tag (no-op if absent) |
| `list <id>` | List all tags for a prompt, one per line |

**Status:** Planned | **SRS:** REQ-F-128–130

---

### `pp export`

Exports prompts or a subtree to a new library directory.

Options:

- `--out <path>`: destination (required).
- `--folder <path>`: filter by folder.
- `--root <path>`: library root.

Refuses to overwrite an existing directory.

Planned extensions:

- `--category <name>`: filter by category.
- Compressed archive output.

**Status:** Implemented | **SRS:** REQ-F-071–075

---

### `pp import <source-path>`

Imports prompts from another library folder or export directory.

Options:

- `--on-conflict skip|replace`: conflict strategy (default: `skip`).
- `--root <path>`: library root.

Source must contain `.promptlib/index.tsv`.

Planned extension:

- `rename` conflict strategy.

**Status:** Implemented | **SRS:** REQ-F-076–080

---

### `pp backup`

Creates a restorable full-library copy (equivalent to unfiltered export).

Options:

- `--out <path>`: destination (required).
- `--root <path>`: library root.

**Status:** Implemented | **SRS:** REQ-F-081

---

### `pp reindex`

Rebuilds `index.tsv` from per-prompt `metadata.tsv` files (the source of truth).

Options:

- `--root <path>`: library root.
- `--dry-run`: print differences without writing.

**Status:** Planned | **SRS:** REQ-F-131–132

---

### `pp stats`

Displays library-wide statistics: prompt count, folder count, category count,
unique tag count, versioned snapshot count, archived prompt count, and disk usage.

Options:

- `--json`: output as JSON object.

**Status:** Planned | **SRS:** REQ-F-133

## Environment

- `PROMPTLIB_ROOT` points to the active library root.
- If unset, the application uses auto-discovery (walk-up from cwd), then falls
  back to the user's home directory.

## Validation

- Folder paths cannot be absolute paths and cannot contain `..` or `:`.
- Metadata fields reject tabs and newlines.
- Tags reject commas because comma is used for the index representation.
- Destructive folder/category and prompt delete operations require explicit `--yes`.
